1. Authentication Overview
1.1 Supported Authentication Methods
The ZuriTrails Flutter app supports four authentication flows:

Simple Email Registration - Quick signup with basic info (first name, last name, birthday, email, password)
Multi-Step Registration - Comprehensive 13-step onboarding flow
Phone Number + OTP - SMS verification with 6-digit code
Social Login - Google, Apple, Facebook
1.2 Standard API Response Format
Success Response:

{
  "success": true,
  "message": "Operation successful",
  "data": { ... },
  "meta": {
    "timestamp": "2026-01-15T15:31:18+03:00",
    "version": "1.0.0"
  }
}
Error Response:

{
  "success": false,
  "message": "Validation failed",
  "errors": {
    "email": ["The email has already been taken."]
  },
  "meta": {
    "timestamp": "2026-01-15T15:31:18+03:00",
    "version": "1.0.0"
  }
}
2. Database Schema
2.1 Users Table
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    
    -- Authentication
    email VARCHAR(255) UNIQUE,
    phone_number VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255),
    
    -- Basic Profile (from simple signup)
    first_name VARCHAR(100) NOT NULL,
    last_name VARCHAR(100) NOT NULL,
    display_name VARCHAR(100),
    date_of_birth DATE,
    
    -- Extended Profile (from multi-step signup)
    bio TEXT,
    photo_url VARCHAR(500),
    gender VARCHAR(20),
    nationality VARCHAR(100),
    home_country VARCHAR(100),
    referral_source VARCHAR(50),
    
    -- ID Documents
    id_number VARCHAR(50),
    passport_number VARCHAR(50),
    
    -- Address
    address TEXT,
    city VARCHAR(100),
    country VARCHAR(100) DEFAULT 'Kenya',
    postal_code VARCHAR(20),
    
    -- Preferences (JSONB arrays)
    interests JSONB DEFAULT '[]',
    languages JSONB DEFAULT '[]',
    travel_styles JSONB DEFAULT '[]',
    activity_preferences JSONB DEFAULT '[]',
    dietary_restrictions JSONB DEFAULT '[]',
    accessibility_needs JSONB DEFAULT '[]',
    
    -- Settings
    preferred_currency VARCHAR(3) DEFAULT 'KES',
    email_notifications BOOLEAN DEFAULT true,
    sms_notifications BOOLEAN DEFAULT false,
    push_notifications BOOLEAN DEFAULT true,
    location_enabled BOOLEAN DEFAULT false,
    show_distance_away BOOLEAN DEFAULT true,
    
    -- Emergency Contact
    emergency_contact_name VARCHAR(100),
    emergency_contact_phone VARCHAR(20),
    emergency_contact_relation VARCHAR(50),
    
    -- Travel Preferences
    travel_style VARCHAR(50),
    travel_insurance BOOLEAN DEFAULT false,
    
    -- Gamification
    explorer_level VARCHAR(20) DEFAULT 'explorer',
    current_xp INTEGER DEFAULT 0,
    streak_days INTEGER DEFAULT 0,
    unlocked_badges JSONB DEFAULT '[]',
    
    -- Account Settings
    is_profile_public BOOLEAN DEFAULT true,
    share_location_with_friends BOOLEAN DEFAULT false,
    
    -- Verification Status
    email_verified_at TIMESTAMP,
    phone_verified_at TIMESTAMP,
    
    -- Metadata
    last_active_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    deleted_at TIMESTAMP
);
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_phone ON users(phone_number);
CREATE INDEX idx_users_level ON users(explorer_level);
CREATE INDEX idx_users_created ON users(created_at);
2.2 Personal Access Tokens (Laravel Sanctum)
CREATE TABLE personal_access_tokens (
    id BIGSERIAL PRIMARY KEY,
    tokenable_type VARCHAR(255) NOT NULL,
    tokenable_id UUID NOT NULL,
    name VARCHAR(255) NOT NULL,
    token VARCHAR(64) UNIQUE NOT NULL,
    abilities TEXT,
    last_used_at TIMESTAMP,
    expires_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_tokens_tokenable ON personal_access_tokens(tokenable_type, tokenable_id);
CREATE INDEX idx_tokens_token ON personal_access_tokens(token);
2.3 Social Providers Table
CREATE TABLE social_providers (
    id BIGSERIAL PRIMARY KEY,
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    provider VARCHAR(50) NOT NULL,
    provider_id VARCHAR(255) NOT NULL,
    provider_token TEXT,
    provider_refresh_token TEXT,
    provider_email VARCHAR(255),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(provider, provider_id)
);
CREATE INDEX idx_social_providers_user ON social_providers(user_id);
CREATE INDEX idx_social_providers_email ON social_providers(provider_email);
2.4 Phone Verifications Table
CREATE TABLE phone_verifications (
    id BIGSERIAL PRIMARY KEY,
    phone_number VARCHAR(20) NOT NULL,
    otp_code VARCHAR(6) NOT NULL,
    verification_id VARCHAR(50) UNIQUE NOT NULL,
    is_verified BOOLEAN DEFAULT false,
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_phone_verifications_phone ON phone_verifications(phone_number);
CREATE INDEX idx_phone_verifications_id ON phone_verifications(verification_id);
3. Simple Email Registration
Used in: 
signup_screen.dart

This is the basic signup flow with minimal required fields.

3.1 Endpoint
POST /api/v1/auth/register

3.2 Request Body
{
  "first_name": "Chris",
  "last_name": "Mutwiri",
  "date_of_birth": "1995-06-15",
  "email": "chris@example.com",
  "password": "SecurePass123!",
  "password_confirmation": "SecurePass123!"
}
3.3 Validation Rules
[
    'first_name' => 'required|string|max:100',
    'last_name' => 'required|string|max:100',
    'date_of_birth' => 'required|date|before:today|after:' . now()->subYears(100)->toDateString(),
    'email' => 'required|email|unique:users,email',
    'password' => 'required|string|min:8|confirmed'
]
Password Requirements:

Minimum 8 characters
Must match password_confirmation
No specific complexity requirements in simple signup
Age Validation:

Must be at least 18 years old
Date must be before today
Date must be after 1900

3.5 Error Responses
Email Already Taken (422):

{
  "success": false,
  "message": "Validation failed",
  "errors": {
    "email": ["The email has already been taken."]
  }
}
Underage User (422):

{
  "success": false,
  "message": "Validation failed",
  "errors": {
    "date_of_birth": ["You must be at least 18 years old to sign up."]
  }
}
3.6 Backend Processing Steps
Validate all fields
Check age requirement (18+)
Hash password using bcrypt
Create user record in users table
Set display_name as first_name + last_name
Set default values:
explorer_level = 'explorer'
current_xp = 0
streak_days = 0
profile_completion = 25
Generate access token (Laravel Sanctum)
Send welcome email with verification link
Return user object + token
4. Multi-Step Registration (13 Steps)
Used in: 
multi_step_signup/signup_flow_coordinator.dart

This is the comprehensive onboarding flow that collects detailed user preferences.

4.1 Signup Flow Steps
Step 1: Welcome Step - Introduction
Step 2: Birthday Step - Date of birth
Step 3: Referral Step - How they found the app
Step 4: Name Step - First and last name
Step 5: Gender Step - Gender selection
Step 6: Country Step - Home country
Step 7: Travel Style Step - Travel preferences
Step 8: Interests Step - Activity interests
Step 9: Profile Photo Step - Photo upload (optional)
Step 10: Notifications Step - Push notification preferences
Step 11: Activity Preferences Step - Preferred activities
Step 12: Location Step - GPS permission
Step 13: Privacy Step - Terms acceptance

4.2 Endpoint
POST /api/v1/auth/register/multi-step

{
  "first_name": "Chris",
  "last_name": "Mutwiri",
  "birthday": "1995-06-15",

  "password": "SecurePass123!",
  "password_confirmation": "SecurePass123!"

4.4 Validation Rules
[
    // Step 4: Name
    'first_name' => 'required|string|max:100',
    'last_name' => 'required|string|max:100',
    
    // Step 2: Birthday
    'birthday' => 'required|date|before:today|after:1900-01-01',
    
    // Step 3: Referral Source
    'referral_source' => 'nullable|string|in:google_search,social_media,friend,advertisement,app_store,other',
    
    // Step 5: Gender
    'gender' => 'required|in:male,female,other,prefer_not_to_say',
    
    // Step 6: Home Country
    'home_country' => 'required|string|max:100',
    
    // Step 7: Travel Styles (multi-select, minimum 1)
    'travel_styles' => 'required|array|min:1',
    'travel_styles.*' => 'string|in:adventure,luxury,budget,eco_friendly,cultural,solo,group,family,backpacking',
    
    // Step 8: Interests (multi-select, minimum 1)
    'interests' => 'required|array|min:1',
    'interests.*' => 'string|in:wildlife,photography,hiking,culture,food,beaches,history,nature,adventure,relaxation,nightlife,shopping,sports',
    
    // Step 9: Profile Photo (optional)
    'profile_photo_path' => 'nullable|string',
    
    // Step 10: Notifications
    'notifications_enabled' => 'required|boolean',
    
    // Step 11: Activity Preferences (multi-select, minimum 1)
    'activity_preferences' => 'required|array|min:1',
    'activity_preferences.*' => 'string|in:safaris,hiking,city_tours,beach_activities,cultural_visits,food_tours,photography,wildlife_watching,water_sports,mountain_climbing',
    
    // Step 12: Location
    'location_enabled' => 'required|boolean',
    'show_distance_away' => 'required|boolean',
    
    // Email & Password
    'email' => 'required|email|unique:users,email',
    'password' => 'required|string|min:8|confirmed'
]

6. Phone Authentication
Used in: 
phone_auth_screen.dart
 + 
verification_code_screen.dart

6.1 Step 1: Request OTP
POST /api/v1/auth/phone/request-otp

Request:

{
  "phone_number": "+254712345678",
  "country_code": "KE",
  "is_signup": true
}
Validation:

[
    'phone_number' => 'required|string|regex:/^\+[1-9]\d{1,14}$/',
    'country_code' => 'required|string|size:2',
    'is_signup' => 'required|boolean'
]
Success Response (200 OK):

{
  "success": true,
  "message": "Verification code sent to +254712345678",
  "data": {
    "verification_id": "ver_abc123def456",
    "phone_number": "+254712345678",
    "expires_at": "2026-01-15T15:36:18+03:00",
    "resend_available_at": "2026-01-15T15:32:18+03:00"
  }
}
Backend Processing:

Validate phone number format (E.164)
If is_signup = true:
Check if phone already registered
Return error if exists
If is_signup = false:
Check if phone exists in database
Return error if not found
Generate 6-digit OTP code
Create record in phone_verifications table:
verification_id = UUID
otp_code = 6-digit code
expires_at = NOW() + 5 minutes
Send SMS via Africa's Talking / Twilio
Return verification_id
6.2 Step 2: Verify OTP
POST /api/v1/auth/phone/verify-otp

Request:

{
  "verification_id": "ver_abc123def456",
  "otp_code": "123456",
  "is_signup": true,
  "signup_data": {
    "first_name": "Chris",
    "last_name": "Mutwiri",
    "travel_styles": ["adventure"],
    "interests": ["wildlife"]
  }
}
Validation:

[
    'verification_id' => 'required|string|exists:phone_verifications,verification_id',
    'otp_code' => 'required|string|size:6',
    'is_signup' => 'required|boolean',
    'signup_data' => 'required_if:is_signup,true|array',
    'signup_data.first_name' => 'required_if:is_signup,true|string|max:100',
    'signup_data.last_name' => 'required_if:is_signup,true|string|max:100',
    'signup_data.travel_styles' => 'nullable|array',
    'signup_data.interests' => 'nullable|array'
]

Error Responses:

Invalid OTP (422):

{
  "success": false,
  "message": "Invalid verification code",
  "errors": {
    "otp_code": ["The verification code is incorrect."]

}
Expired OTP (422):

{
  "success": false,

Find verification record by verification_id
Check if expired
Verify OTP code matches

Create new user with phone_number

Store signup_data if provided
Generate token
Return user + token with requires_onboarding = true
If is_signup = false:
Find user by phone_number
Update last_active_at
Generate token
Return user + token
